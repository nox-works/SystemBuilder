# SystemBuilder — The System Architect

**Engineers complete, production-grade systems from any topic, idea, or knowledge domain.**

From a compendium to a course, from a playbook to a framework, from an encyclopedia to a toolkit — SystemBuilder takes raw knowledge and architects it into structured, navigable, usable systems. Built on the **Veritas lineage** of first-principles thinking, with the polymath breadth to design any system type and the engineering rigor to build it.

## System Types

| Type | Best For | Output |
|------|----------|--------|
| **Compendium** | Comprehensive domain reference | Multi-chapter volume with navigation |
| **Course** | Teaching skills end-to-end | Curriculum with objectives, assessments, assets |
| **Playbook** | Tactical response, crisis | Decision trees, protocols, quick reference |
| **Framework** | Conceptual understanding | Abstracted model with components, relationships |
| **Reference Architecture** | System design, standards | Blueprint with layers, interfaces, implementation guide |
| **Encyclopedia** | Quick lookup, terminology | Structured entries with cross-references |
| **Toolkit** | Repeated practical tasks | Instruments, templates, examples |
| **Manifesto** | Worldviews, founding philosophies | Principles, argument, implications, action |
| **Knowledge Graph** | Complex non-linear domains | Interconnected concept map with navigation paths |
| **Hybrid** | Complex multi-need systems | Combined types in unified architecture |

## Pipeline

1. **Reception & Domain Framing** — What is this system for? Who is it for?
2. **First-Principles Decomposition** — Strip the domain to fundamentals
3. **Multi-Perspective Scan** — User, knowledge, task, growth, constraint, integration, experience
4. **System Type Selection & Architecture** — Choose type, design structure
5. **Component Decomposition & Build Plan** — Break into buildable units
6. **Knowledge Synthesis** — Generate deep knowledge per component
7. **System Construction** — Build everything with navigation and cross-references
8. **Validation** — Check completeness, coherence, usability
9. **Delivery** — Present with architecture rationale and usage guide

## Usage

```bash
/system build a compendium on negotiation
/system build a course on persuasion @course
/system build a playbook for crisis @playbook
/system architect a framework for decision-making @tier1
```

## Quality Guarantees

- **Coherent**: Every part serves the whole
- **Navigable**: Users always know where they are and where to go
- **Complete**: Self-contained — everything needed is inside
- **Modular**: Components are replaceable without breaking the system
- **Minimal**: Nothing unnecessary
