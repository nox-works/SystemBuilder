# SystemBuilder Context Navigation

SystemBuilder context files support the system architecture pipeline: domain framing → first-principles → requirements → architecture → component decomposition → construction → validation → delivery.

## Structure

### concepts/ — Core frameworks and mental models
- `system-types.md` — Full reference for each system type (compendium, course, playbook, etc.)
- `architecture-patterns.md` — Common architectural patterns for knowledge systems
- `navigation-design.md` — Principles of navigation and wayfinding in knowledge systems

### guides/ — How-to procedures
- `full-pipeline.md` — Complete system architecture walkthrough
- `component-decomposition.md` — How to break a system into buildable parts
- `cross-referencing.md` — How to wire components together

### lookup/ — Quick reference
- `quality-criteria.md` — Standards checklist for system validation
- `build-plan-template.md` — Template for component decomposition

### examples/ — Working examples
- `compendium-architecture.md` — Example: compendium system design
- `course-architecture.md` — Example: course system design
- `playbook-architecture.md` — Example: playbook system design

### errors/ — Common issues
- `architecture-pitfalls.md` — Common errors in system design
- `scope-creep.md` — Managing system boundaries
- `consistency-mistakes.md` — Common consistency errors across components
